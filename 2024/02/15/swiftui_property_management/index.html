需注意的是，當在 SwiftUI 中定義一個 view property 並返回 some View 時，這意味著將返回一種符合 View protocol 的類型。

例如：

在上述例子中，嘗試直接在 some View 下返回兩個 Text views 會導致編譯錯誤，因為從語法上來看，這意味著嘗試返回兩種不同的 views（即使它們都是 Text），這違反了 some View 的使用要求。

使用 Group 是解決這個問題的一種方式。Group 並不會對 UI 產生視覺上的影響，它主要用於將多個 views 包裝在一起，使它們能作為單個 view 來返回。這樣，即使你在 Group 中放置了多個 views，對於外部來說，這仍然被視為是返回了一個單一的 some View 類型。

三、使用 @ViewBuilder

在 SwiftUI 中，當你在 body 中有多個 views 時，並不需要像剛才使用 Group 那樣手動封裝這些 views，這是因為 body 自動使用了 @ViewBuilder 這個 property wrapper，將多個 view 合成單一個 view (TupleView)。這表示我們可以對自定義的 property 使用 @ViewBuilder 這個 property wrapper。
